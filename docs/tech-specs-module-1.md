# Техническое задание — Модуль 1: Парсер-менеджер

## 1. Назначение модуля

Модуль 1 является управляющим ядром системы JobRadar. Он не собирает вакансии сам — он принимает задачи на сбор, валидирует их, хранит в базе данных и предоставляет очередь задач для краулера (Модуль 2).

---

## 2. Функциональные требования

| ID | Требование |
|---|---|
| FR-01 | Система принимает одну поисковую фразу и источник, создавая одну задачу |
| FR-02 | Система валидирует источник — допустимы только значения из разрешённого списка |
| FR-03 | Система не создаёт дублирующие задачи: если задача с таким же ключевым словом и источником уже активна (статус NEW или IN_PROGRESS), новая не создаётся |
| FR-04 | Система позволяет просматривать список задач с фильтрацией по количеству записей |
| FR-05 | Система позволяет менять статус задачи |
| FR-06 | Система логирует ключевые события и ошибки |
| FR-07 | Система предоставляет CLI-интерфейс, в том числе интерактивный режим |

---

## 3. Формат входных данных

```json
{
  "keyword": "python developer",
  "source": "habr"
}
```

Система создаёт **одну задачу** на одну поисковую фразу.

### Ограничения на входные данные

| Поле | Тип | Ограничения |
|---|---|---|
| `keyword` | строка | Непустая строка, приводится к нижнему регистру, пробелы обрезаются |
| `source` | строка | Только из разрешённого списка: `habr`, `hh`, `geekjob` |

---

## 4. Интерфейс взаимодействия (CLI)

### 4.1 Команды

| Команда | Описание | Пример |
|---|---|---|
| `init` | Инициализация базы данных | `job_radar init` |
| `add <keyword> <source>` | Создать задачу по поисковой фразе | `job_radar add "python developer" habr` |
| `list [limit]` | Показать последние задачи | `job_radar list 10` |
| `interactive` | Запустить интерактивный режим | `job_radar interactive` |

### 4.2 Пример работы команды `add`

```
$ job_radar add "python developer" habr

✔ Задача создана! ID: 1 | Ищем: python developer @ habr
```

---

## 5. Обработка ошибок

| Ситуация | Поведение системы |
|---|---|
| Неизвестный источник | Вывод сообщения об ошибке, задача не создаётся |
| Пустое ключевое слово | Отказ с пояснением, остальные слова обрабатываются |
| Дублирующая задача | Сообщение с ID существующей задачи, новая задача не создаётся |
| Ошибка БД | Вывод ошибки, программа не падает |

---

## 6. Статусы задач

| Статус | Описание |
|---|---|
| `NEW` | Задача создана, ожидает краулера |
| `IN_PROGRESS` | Краулер взял задачу в работу |
| `COMPLETED` | Сбор завершён успешно |
| `FAILED` | Краулер завершил с ошибкой |
| `CANCELLED` | Отменена пользователем |
| `EXPIRED` | Устарела (не взята в работу слишком долго) |
| `RETRIED` | Повторно запущена после неудачи |

---

## 7. Ограничения и допущения

- База данных: SQLite (файл `job_radar.db` в корне проекта)
- Язык: Python 3.10+
- Авторизация отсутствует
- Сетевые запросы в данном модуле не выполняются
- Модуль не запускает краулер — только управляет очередью задач
